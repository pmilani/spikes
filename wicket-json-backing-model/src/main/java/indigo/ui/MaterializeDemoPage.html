Showing a panel with components backed by materialized beans

Materializing is an object mapping technique, wherein your Java model is defined purely by interfaces with accessor methods.
When deserializing, Jackson can materialize those interfaces, which means creating on the fly a concrete instance that implements them.
Setting the Jackson object mapper to ignore unknown properties can lead to clean and concise object models to access only the data that will be used.
